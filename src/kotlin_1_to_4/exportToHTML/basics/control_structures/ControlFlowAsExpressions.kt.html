5    // In Java the control blocks are statements, i.e. they don't return anything in it's evaluations think boolean 
6     
7    // So in Java it's common to have: 
8     
9    /* 
10   * boolean someBoolean = null; if (something) { someBoolean = assignThisThing } else { someBoolean = assignOtherThing } . 
11   * 
12   * */ 
13    
14   // In Kotlin, control blocks are expressions and so the result can be directly assigned to a value or passed as 
15   // an argument to another function 
16    
17   // Allows boilerplate code to be reduced and made more readable 
18    
19   val date = Date() 
20    
21   // Use of control flow expression to assign a boolean! 
22    
23   val today = if (date.time == 19L) true else false // Or something like this ... 
24    
25   fun checkIfZero(x: Int): Boolean 
26   { 
27       return if (x == 0) true else false 
28    
29       // Compare with: if (x == 0) { return true } else { return false } ... in Java 
30   } 
31    
32   // Same concept can be applied to try/catch blocks in Kotlin 
33
