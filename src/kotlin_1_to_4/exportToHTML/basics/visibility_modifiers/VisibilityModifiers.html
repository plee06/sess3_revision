VisibilityModifiers 
1    Just like in Java, Kotlin offers up visibility modifiers for making certain code visible and others not. 
2     
3    In Kotlin there are four visibility modifiers: public, internal, protected, and private. 
4     
5    When no modifier is given, the "default" is used which is public. 
6     
7    Again the four visibility modifiers in Kotlin are: 
8     
9    1. public 
10   2. internal 
11   3. protected 
12   4. private 
13    
14   private -> can only be accessed in the same file, or if it's a class it is only accessible in that class! 
15   protected -> functions / properties declared as protected inside a class / interface only visible to members 
16   of that class or interface as well as subclasses 
17    
18   Wait, pause. So what's the difference between private and protected? 
19    
20   private => MUST BE SAME EXACT CLASS 
21   protected => AVAILABLE TO SUBCLASSES! 
22    
23   internal -> the whole point of internal is to act like "public" but instead of being "public" to everybody 
24   it is only public to the same module, i.e. think of a package. 
25    
26   "public to module, not public to the universe..."
